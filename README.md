# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
가상의 '반려동물 건강 일지'


## 2. 시나리오 (5~10줄)
반려동물 건강 일지 프로그램인 **'멍냥 헬스케어'**는 우리의 소중한 가족인 반려동물의 상태를 체계적으로 관리하는 프로그램입니다.사용자는 먼저 반려동물의 이름과 종, 목표 체중을 설정하며 시작합니다. 매일 사료 섭취량과 산책 시간(또는 활동량)을 입력하면, 프로그램은 조건문을 통해 권장량 대비 상태가 '적정'한지 아니면 '에너지 과잉'인지를 즉각 판정해 줍니다. 메인 메뉴에서는 새로운 기록을 추가하거나 그동안 쌓인 일지를 조회할 수 있고, 배열에 저장된 최근 데이터들을 바탕으로 함수가 평균 활동량을 계산해 건강 점수를 산출합니다. 단순히 점수를 매기는 성적 관리에서 벗어나, 생동감 있는 데이터로 반려동물의 건강 패턴을 한눈에 파악할 수 있는 시스템을 완성할 계획입니다.


<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1.오늘의 건강 상태 기록 (입력 및 배열 저장)
2. 실시간 건강 상태 진단 (조건문 활용)
3. 최근 7일 건강 요약 보고서 (함수 및 계산)
4.날짜별 기록 검색 (배열 탐색)
5. 목표 체중 달성도 확인 (수치 연산)
6. 프로그램 종료 및 데이터 초기화 (반복문 제어)

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 시나리오 기획
개발내용
동물의 이름,종,하루권장 산책시간(정수형,분 단위) 등의 기초데이터 설정.
조건문을 이용하여 입력된 데이터를 바탕으로 건강상태 판별.
반복문 및 메뉴를 이용하여 사용자가 종료를 선택하기 전까지 반복.
함수 및 배열을 통해 데이터를 구조적으로 저장하고 관리.

### **🤖 AI 파트너십 과정**
프롬프트 내용:학교 과제의 조건(4단계 도구 활용)을 이용한 독창적이고 창의적인 주제 추천
적용내용:성적 관리 프로그램의 논리를 유지하면서도 독창적인 3가지 테마(건강,키오스크,식재료)를 제안하여 주제를 제시해줌.

프롬프트 내용:C언어 4단계 과정 로드맵(입출력->조건문->반복문->배열/함수)에 적합한 '반려동물 건강일지'라는 로드맵을 요청함.
적용내용:오늘의 건강 상태 기록 (입력 및 배열 저장)->실시간 건강 상태 진단 (조건문 활용)->최근 7일 건강 요약 보고서 (함수 및 계산)->날짜별 기록 검색 (배열 탐색)->목표 체중 달성도 확인 (수치 연산)->프로그램 종료 및 데이터 초기화 (반복문 제어)의 로드맵 뼈대 확립.

### **Troubleshooting&기술 회고**
문제1:input()으로 받은 값은 항상**문자열(str)** 입니다. 이를 숫자로 변환하지 않고 연산하면 에러가 발생한다.
해결:int()또는 float()함수를 사용해 명시적으로 형변환을 해줘야한다.

문제2:4단계 배열(리스트) 기능에서, 저장된 기록이 2개인데 5번째 기록을 검색하려고 할 때 발생한다.
해결:if문을 사용하여 사용자가 입력한 인덱스가 len(health_1ogs) 범위 내에 있는지 확인해야한다.

문제3: 함수 내부에서 메인 루프에 있는 리스트를 수정하려고 할 때, 리스트가 인식되지 않거나 초기화되는 경우가 생긴다.
해결: 함수를 정의할 때 매개변수(Parameter)로 리스트를 전달받아 처리하거나, global 키워드의 개념을 이해하고 적용해야한다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 특별 칭호를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 용사의 이름, 체력, 마력, 공격력을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 종합 전투력 산출: (체력 * 0.7) + (마력 * 0.5) + (공격력 * 1.8).
  * if-elif-else를 사용하여 S등급부터 F등급까지 5단계 등급 판정 로직 추가.
  * and 연산자를 활용하여 '전설의 대마법사' 등 특별 칭호 부여 기능 구현.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
